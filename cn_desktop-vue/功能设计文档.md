# Clone-NPM Desktop Vue — 功能设计文档

## 一、项目概述

**项目名称**：Clone-NPM Desktop Vue  
**技术栈**：Vue 3 + Electron + Vite + Element Plus + Pinia + node-pty + xterm.js  
**项目定位**：一款面向前端开发团队的 **桌面端项目管理工具**，核心功能是批量管理 Git 仓库的 Clone、依赖安装、分支切换、IDE 打开、开发服务启动等日常操作，同时集成了 AI 模型管理和系统入口导航。

---

## 三、核心功能模块

### 3.1 项目配置管理（ConfigPage）

这是应用的**主页面**，提供完整的项目仓库管理能力。

#### 3.1.1 项目配置项

每个项目配置项（`ConfigItem`）包含以下字段：

| 字段 | 说明 |
|------|------|
| `enable` | 是否勾选执行（批量操作时使用） |
| `pName` | 项目分组名称（如"组件库"、"山东黄金地勘"） |
| `projectName` | 系统名称（如"鹰眼智展大屏"） |
| `gitUrl` | Git 仓库地址 |
| `branch` | 目标分支（默认 master） |
| `savePath` | 本地保存路径 |
| `codePath` | 代码子路径（如 `\webapp`） |
| `testEnvUrl` | 测试环境地址 |
| `prodEnvUrl` | 生产环境地址 |

#### 3.1.2 五大操作按钮

| 操作 | 功能 | 说明 |
|------|------|------|
| **Clone + 分支 + IDE** | 克隆仓库 → 切换分支 → 用 IDE 打开 | 一键完成仓库初始化 |
| **清除 + 重装** | 删除 node_modules → 清除缓存 → npm install | 解决依赖问题 |
| **IDE 打开** | 用配置的 IDE 打开项目目录 | 快速进入开发 |
| **拉取 + 启动** | git pull → npm run dev | 更新代码并启动开发服务器 |
| **一键完成** | Clone → 切分支 → IDE 打开 → npm install | 全流程自动化 |

支持**单个项目操作**和**批量操作**（勾选多个项目后点击顶部按钮）。

#### 3.1.3 IDE 配置

- 支持添加多个 IDE 编辑器路径（VS Code、Trae、Cursor、CodeBuddy 等）
- 下拉选择当前使用的 IDE
- 通过文件选择器添加新的编辑器

#### 3.1.4 项目分组

- 左侧边栏按 `pName` 自动分组
- 支持按分组筛选项目
- 显示已勾选执行的项目列表，点击可快速定位

---

### 3.2 终端面板（TerminalPanel）

基于 **node-pty + xterm.js** 实现的内置终端系统。

#### 功能特性

- **PTY 终端**：使用 `node-pty` 创建真实的伪终端，支持完整的命令行交互
- **实时输出**：Git/NPM 命令的输出实时显示在终端中
- **日志着色**：不同类型的日志用不同颜色区分（成功=绿色、错误=红色、警告=黄色）
- **交互式操作**：用户可以直接在终端中输入命令
- **终端管理**：支持清空、关闭、自动调整大小

#### IPC 通信

```
渲染进程 → 主进程：
  - terminal-write: 向终端写入数据
  - terminal-resize: 调整终端大小
  - terminal-close: 关闭终端

主进程 → 渲染进程：
  - terminal-data: 终端输出数据
  - terminal-exit: 终端进程退出
```

### 3.3 AI 模型管理（AIModelsPage）

#### 3.3.1 多 API 配置

- 支持配置多个 AI API（如 OpenAI 兼容接口）
- 每个 API 包含：名称、URL、Key
- 支持添加、删除、切换 API
- 配置自动保存到 localStorage

#### 3.3.2 模型查询与分类

- 通过 `/models` 接口查询可用模型列表
- 自动按厂商分类：GPT、Claude、Gemini、通义、DeepSeek、其他
- 显示各分类的模型数量
- 点击模型 ID 可快速复制

#### 3.3.3 AI 对话

- 支持选择模型进行对话
- **流式输出**（SSE）：实时显示 AI 回复
- 支持多轮对话上下文
- 错误处理与友好提示
- 启用 web_search 参数支持联网搜索

---

### 3.4 AI 平台导航（AIChatPage）

提供国内外主流 AI 平台的快速入口：

**国产 AI 平台**：文心一言、通义千问、智谱清言、讯飞星火、腾讯混元、Kimi、豆包、DeepSeek

**国外 AI 平台**：ChatGPT、Claude、Gemini、Perplexity、Copilot、Mistral

点击卡片即可在系统浏览器中打开对应平台。

---

### 3.5 系统入口（IframePage）

基于 iframe 的多标签页系统入口管理器。

#### 功能

- 内置默认系统：禅道、GitLab、知识库、掘金
- 支持新增、编辑、删除系统标签
- 在应用内通过 iframe 嵌入显示系统页面
- 配置保存到 `config.json` 的 `systemTabs` 字段
- 配置了宽松的 Cookie 策略，支持 iframe 内的登录态保持

## 四、Electron 主进程功能

### 4.1 窗口管理

- 默认窗口尺寸：1200×800，最小 900×600
- 开发模式自动打开 DevTools
- 深色背景主题（`#1a1a2e`）

### 4.2 配置文件管理

配置文件查找优先级：
1. 用户数据目录（`app.getPath('userData')/config.json`）
2. extraResources 目录（打包后首次运行会复制到用户目录）
3. 项目根目录（开发模式）

### 4.3 Git 操作（通过 PTY）

| IPC 事件 | 功能 | 实现方式 |
|----------|------|----------|
| `git-clone` | 克隆仓库 | `git clone <url> <name>` |
| `git-switch-branch` | 切换分支 | `git fetch origin && git switch <branch>` |
| `git-pull` | 拉取更新 | `git pull` |

### 4.4 NPM 操作（通过 PTY）

| IPC 事件 | 功能 | 实现方式 |
|----------|------|----------|
| `npm-install` | 安装依赖 | `cd /d <path> && npm i` |
| `npm-run-dev` | 启动开发 | `cd /d <path> && npm run dev` |
| `clear-node-modules` | 清除依赖 | 删除 node_modules、package-lock.json，清理 npm 缓存 |

### 4.5 文件系统操作

| IPC 事件 | 功能 |
|----------|------|
| `select-file` | 文件选择对话框 |
| `select-folder` | 文件夹选择对话框 |
| `open-with-ide` | 用 IDE 打开文件夹 |
| `open-in-explorer` | 在资源管理器中打开 |
| `open-external` | 用系统浏览器打开 URL |
| `delete-folder` | 删除文件夹 |

### 4.6 Cookie 策略

- 修改 `SameSite` 策略为 `None`，允许第三方 Cookie
- 允许 iframe 权限请求，确保嵌入系统可正常登录

## 五、状态管理（Pinia Stores）

### config Store

管理项目配置数据，包括：
- `configList`：项目配置列表
- `CodingEditPath` / `CodingEditPathList`：IDE 路径配置
- `systemTabs`：系统入口标签配置
- `activeFilter`：当前分组筛选
- 计算属性：`totalCount`、`enabledItems`、`filteredItems`、`groupedByPName`

### ai Store

管理 AI 相关配置，包括：
- `apiConfigs`：API 配置列表（持久化到 localStorage）
- `currentApiId`：当前使用的 API
- `models` / `modelCategories`：模型列表与分类
- `messages`：对话消息列表
- `currentModel`：当前选择的模型

### terminal Store

管理终端状态：
- `logs`：日志列表
- `isVisible`：终端面板可见性
- `currentTermId`：当前活跃的终端 ID

## 七、UI 设计

### 设计风格

- **深色主题**：主色调 `#1a1a2e`（背景）、`#16213e`（卡片/面板）
- **渐变色**：标题使用 `linear-gradient(135deg, #667eea, #764ba2)` 紫蓝渐变
- **圆角卡片**：`border-radius: 12px`
- **半透明边框**：`rgba(255,255,255,0.1)`

### 组件库

使用 **Element Plus** 组件库，包括：
- 按钮、输入框、选择器、标签页、对话框、表单、复选框、消息提示等
- 图标使用 `@element-plus/icons-vue`

## 八、构建与打包

### 开发

```bash
npm run dev        # 启动 electron-vite 开发服务器
```

### 构建

```bash
npm run build      # 构建渲染进程和主进程
npm run package    # 构建并打包为 Windows 安装程序（NSIS）
```

### 打包配置

- 输出目录：`dist/`
- 安装程序：NSIS（支持自定义安装目录）
- `config.json` 作为 extraResources 打包
- `node-pty` 解包到 asar 外部（避免原生模块兼容问题）

## 十、适用场景

1. **前端团队批量管理多个 Git 仓库**：一键 Clone、安装依赖、启动开发
2. **多项目并行开发**：快速切换不同项目的开发环境
3. **统一 IDE 配置**：团队成员共享编辑器配置
4. **AI 辅助开发**：内置 AI 对话和平台导航
5. **内部系统快速访问**：通过 iframe 嵌入禅道、GitLab 等内部系统
